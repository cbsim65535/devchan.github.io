# 🧠 왜 메모리 중심 사고가 중요한가?

<div style="display: flex;">

<div style="flex: 1;">

### 📌 핵심 개념
- 메모리는 **설계의 중심**
- 성능과 안정성은 **메모리 흐름 이해**에서 출발
- 추상화에만 의존하지 말고, **저수준 원리 인식** 필요

### ⚠️ 주요 문제
- 누수 / 잘못된 참조  
- 단편화 / 오버플로 / GC 지연  
- 동기화 실패 / 캐시 미스

### 🛠 대응
- `Valgrind`, `GDB` 등 도구 활용  
- `Memory Pool`, `Locality` 최적화

</div>

<div style="flex: 1;">


### 🧭 사고 전환
- 성능 이슈는 대부분 **메모리로 귀결**
- “이 코드는 어떤 메모리를 언제 쓰는가?”

### 🧬 개발자의 자세
- 추상화된 언어도 **메모리 모델을 이해**해야  
- **버그 해결 = 흐름 추적 능력**

> 💡 메모리를 이해해야  
> 복잡한 시스템을 설계하고 유지할 수 있다.
